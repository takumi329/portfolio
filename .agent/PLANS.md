# 実行プラン (ExecPlans)

複雑な機能追加や大規模なリファクタリングを行う際は、以下のフォーマットに従って `plans/` ディレクトリ内に Markdown ファイル（例: `plans/auth-feature.md`）としてプランを作成すること。作成後は必ず `codex-review` スキルでレビューを受けること。

## テンプレート

```markdown
# [機能名・タスク名] 実行プラン

## 概要
何を実装するのか、何のための変更かを簡潔に記載。

## 影響範囲
どのファイル、モジュールに変更が加わるか。

## ステップバイステップ実装手順
1. **[ステップ1]**: 具体的な実装内容
2. **[ステップ2]**: 具体的な実装内容
   - サブタスク
3. **[ステップN]**: テスト・UI確認

## 確認事項
- [ ] 完了条件1
- [ ] 完了条件2
```
